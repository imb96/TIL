정규표현식은 문자 검색과 교체에 사용되는 패턴으로 강력한 기능을 제공한다.
JavaScript 에선 RegExp 객체와 문자열 메서드를 조합해 정규표현식을 사용할 수 있다.
정규식 객체를 만들 땐 두 가지 문법이 사용된다.
regexp = new RegExp("pattern", "flags");
// or
regexp = /pattern/; // no flags
regexp = /pattern/gmi;  // flags g, m, i

위 예시에서 regexp는 내장 클래스, RegExp의 인스턴스가 된다.
/.../ 를 사용하면 중간에 표현식을 넣을 수 없다. 따라서 슬래시를 사용한 방법은 정적이다.
동적으로 생성된 문자열을 가지고 정규표현식을 만들어야 할 때는 긴 문법을 사용한다.
let tag = prompt("what tag do you want?", "h2");
let regexp = new RegExp(`<${tag}>`); // 프롬프트에서 "h2"라고 대답한 경우, /<h2>/와 동일한 역할을 한다.

정규 표현식은 패턴과 선택적으로 사용할 수 있는 플래그 g,i,m,u,s,y로 구성된다.
g: 패턴과 일치하는 모든 것들을 찾는다. g 플래그가 없다면 패턴과 일치하는 첫 번째 결과만 반환.
i: 대, 소문자 구분없이 검색한다. ex) A === a
m: 다중 행 모드를 활성화 한다.
u: 유니코드 전체를 지원한다.
s: .(점) 이 개행 문자 \n 도 포함하도록 'dotall'모드를 활성화.
y: 문자 내 특정 위치에서 검색을 진행하는 'sticky' 모드를 활성화 시킨다.

\d 는 숫자를 대표하는 정규표현식이다. (d 는 digit를 의미)
\w는 글자를 대표하는 정규표현식이다.
- a,b,가,나,1,2 와 같은 문자와 숫자를 포함한다.
- 특수문자는 포함하지 않지만, _(언더스코어)는 포함한다.
\d는 숫자를 한글자만 찾는다.
연결된 숫자를 찾을 때는 + 를 이용한다.
- + 는 “하나 혹은 그 이상 연결된" 이라는 의미.
- \d+ 는 “하나 혹은 그 이상 연결된 숫자”를 의미
*는 “0개 이상"을 의미.
?는 “있거나 없거나" 를 의미. -?는 “-가 있거나 없거나”
-또는 공백이 있거나 없거나는 [- ]?
{숫자}는 “숫자번 반복한다"는 뜻이다. \d{2}는 “숫자가 연속 두 번 나온다" 는 뜻이다.
알파벳 소문자 모음(a,e,i,o,u)만 고르고 싶을떄 [aeiou]
정규표현식에서 대괄호[]안에 글자를 넣으면 해당 글자를 모두 선택할 수있다.
[a-z]는 “a부터 z까지 글자를 모두 선택하라"는 의미다.
[a-z]+ 는 모든 연속된 영어 소문자를 찾을 수 있다.
한글 단어 찾기 ?
한글의 첫 번째 글자는 가 이고 마지막 글자는 힣 이다. 따라서 한글은 [가-힣]으로 찾을 수 있다.
\s 공백 문자 (스페이스, 탭, 뉴라인_

모든 문자 클래스에는 '반대 클래스’가 있다. 같은 글자로 표기하지만 대문자로 사용한다.
'반대’란 다음 예시들처럼 해당 문자를 제외한 모든 문자에 일치한다는 뜻.
/S 공백 문자를 제외한 문자.
/D 숫자를 제외한 문자.
/W 글자 대표 문자를 제외한 글자들 (특수문자, 공백 등)

.(점)은 '아무 문자'에나 일치한다.
점은 줄 바꿈 문자를 제외한 모든 문자와 일치하는 특별한 문자 클래스다.
